## 1. 프로젝트 소개

### 1.1. 개발배경 및 필요성

부산을 포함한 국내외 주요 도시들은 도심 공실률 증가와 그에 따른 자산 가치 하락이라는 경제적 문제에 직면해 있으며, 이는 지역 상권 침체와 도시 활력 저하로 이어지고 있습니다. 동시에 기후 변화와 도시화로 인한 농업 부지 감소는 새로운 대안적 생산 체계의 필요성을 높이고 있습니다.

기존 농산물 생산지는 대부분 도시 외곽이나 교외에 위치해 소비자에게 전달되기까지 장거리 운송이 필요합니다. 이 과정에서 발생하는 **푸드 마일리지**는 탄소 배출을 증가시킬 뿐 아니라 농산물의 신선도 저하와 유통 비용 상승으로 이어집니다.

도심 내 농업·스마트팜에 대한 수요는 늘고 있으나, 공간 확보의 어려움, 전문 재배 지식 부족, 초기 설치 비용 부담, 수익성 불확실성 등으로 일반 시민과 소규모 운영자의 접근은 여전히 제한적입니다. 따라서 **도심 유휴공간을 활용해 생산·수확·판매를 하나의 플랫폼으로 연결**하고, 지역 주민과 소비자가 함께 참여하는 도심형 스마트팜 생태계가 필요합니다.

### 1.2. 개발 목표 및 주요 내용

생산부터 수확, 판매, 소비까지 **지역 안에서 순환하는 생활권 기반 지역 순환형 스마트팜 플랫폼**을 구축하는 것을 목표로 합니다.

- **AI 공간 매핑 · 수익 예측** — 유휴공간의 구조·조건을 분석해 최적의 스마트팜 모듈 배치를 제안하고, 설치 가능성과 예상 수익을 사전에 시뮬레이션
- **3자 매칭** — 공실 제공자 ↔ 도심 농부를 앱으로 연결하고 계약을 관리
- **지역 직거래 마켓** — 수확물을 생활권 내 소비자에게 직접 판매, 푸드 마일리지 없는 신선 농산물 공급

### 1.3. 세부내용

**[사용자 요구사항]**

| 사용자 | 주요 기능 |
| --- | --- |
| 공실 제공자 | 공간 등록, 예상 수익 확인, 입주 농부 관리 및 계약 체결 |
| 도심 농부 | 선호 공간 추천·매칭, 작물 추천 |
| 소비자 | 신선 농산물 직거래 구매, 생산 이력 조회 |

**[기능 요구사항]**

- **AI 예측** — 면적·위치·일조량 기반 수익 예측 및 작물 추천
- **매칭** — 공실 제공자–농부 매칭·계약 관리, 지역 직거래 마켓플레이스 운영
- **결제·관리** — 소셜 로그인, 농산물 거래 인앱 결제, 수익 정산

### 1.4. 기존 서비스 대비 차별성

**(1) 진입 장벽의 완화**
- 기존: 수억 단위의 초기 구축비와 복잡한 상담 절차로 일반인 접근이 어려움
- 봄동마켓: **'공간 공유' 개념**으로 유휴공간 소유자와 예비 농부를 매칭, 누구나 도심 농업에 참여 가능한 환경 제공

**(2) 농산물 직거래**
- 기존: 대부분 도매단위 거래로 개인 구매가 어려움
- 봄동마켓: 도심 스마트팜 생산 작물을 인근 소비자와 직접 연결하는 **지역 기반 직거래 시스템** 구축. 생산 이력과 재배 공간 정보를 앱에서 확인할 수 있어 신뢰성 확보

### 1.5. 사회적가치 도입 계획

1. **건물주·부동산 관리자** — 방치된 유휴공간을 전환해 공실 손실을 완화. AI 공간 매핑으로 배치도·예상 수익을 즉각 시뮬레이션해 사업 불확실성 제거
2. **창업자·부업 희망자** — 토지 매입 없이 도심 유휴공간을 임대해 소자본으로 즉시 재배를 시작하는 '서비스형 농업' 환경 제공
3. **지역 소비자·공동체** — 초신선 로컬 푸드 소비와 지역 경제 참여. 도심 농장은 생태 교육 현장·주민 참여형 체험 프로그램으로 운영되어 지역 커뮤니티 결속 거점으로 활용
4. **기업·공공기관** — 쇠퇴 상권의 공실 문제를 해결하는 지속 가능한 도시 재생 모델로 도입 가능

### 2.2. 사용 기술

**Frontend**
- React Native — Android · iOS 동시 지원 크로스플랫폼 앱 (공실 탐색, AI 수익 예측 결과 조회, 직거래 마켓 UI)

**Backend**
- Spring Boot (Java) — 회원 관리, 계약 체결, 결제·정산 등 트랜잭션 핵심 비즈니스 로직
- FastAPI (Python) — AI 모델 서빙, Spring Boot와 역할 분리로 AI 연산 요청 고속 처리

**AI / Data**
- Python, PyTorch — 공간 면적·위치 데이터를 학습한 수익 예측 모델
- Gemini API — 작물 추천 및 하드웨어 공간 배치 최적화 청사진 생성
- 3D Point Cloud — 공간 매핑

**협업 · 버전 관리**
- GitHub — `main / dev / feature` 브랜치 전략으로 코드 충돌 최소화
- VS Code — ESLint, Prettier 등으로 코드 스타일 일관성 유지
- Notion / Google 공유 문서 — 요구사항 명세, 회의록, API 문서 통합 관리

**활용한 생성형 AI · AI 코딩 도구**
- **Claude Code** — Superpowers 플러그인, Custom Skills / Subagents, MCP(GitHub·Notion 연동)
- **Codex (ChatGPT)** — 코드 생성 및 리팩토링
- **ChatGPT · Gemini** — 아키텍처·ERD·REST API 명세 검토, 코드 리뷰
- **Figma** — 와이어프레임 · 아이콘 · UI 시안

> 정확한 버전 정보는 개발 환경 확정 후 보강 예정
> ex) React Native - 0.xx, Spring Boot - 3.x, Python - 3.11

### 사전 요구사항
- Node.js, JDK 17+, Python 3.11+

### Frontend (React Native)
```bash
cd frontend
npm install
npx react-native run-android   # 또는 run-ios
```

### Backend (Spring Boot)
```bash
cd backend
./gradlew bootRun
```

### AI Server (FastAPI)
```bash
cd ai-server
pip install -r requirements.txt
uvicorn app.main:app --reload
```
